# 智能门锁系统 使用说明书

基于 ESP32 的多功能智能门锁，支持 RFID 刷卡、指纹识别、密码键盘、蓝牙手机 四种开锁方式，搭配 ESP32-CAM 自动拍照取证、Flask 网页管理后台、微信通知推送。

---

## 一、功能总览

### 四种开锁方式
- **RFID 刷卡** — RC522 射频模块，刷卡即时响应
- **指纹识别** — ZW101 半导体指纹模组，自动唤醒，识别速度快
- **密码输入** — 4x4 矩阵键盘，4位数字密码，按 # 确认
- **蓝牙开锁** — 手机 App 通过经典蓝牙连接 ESP32，发送密码开锁

### 自动拍照记录
- 每次验证（成功或失败）自动触发 ESP32-CAM 拍照
- 照片上传到本地 Flask 服务器，Web 界面可浏览

### Web 管理后台
- **验证记录** — 查看所有开锁/失败记录，含照片、时间、方式
- **白名单管理** — 添加/删除 RFID 卡号和指纹 ID
- **黑名单管理** — 封禁丢失的卡号和指纹，防止未授权访问
- **通知设置** — Server酱微信推送，可按事件类型独立开关
- **密码修改** — 在线修改开门密码，自动同步到 ESP32
- **远程开锁** — 网页一键远程开门
- **CSV 导出** — 导出验证记录为 CSV 文件

### 微信通知
- 接入 Server酱（sct.ftqq.com），开锁/失败/远程开锁事件实时推送到微信

### 云端配置同步
- ESP32 每隔数秒从服务器拉取最新配置（密码、白名单、黑名单）
- WiFi 断线自动重连

### 安全防护
- **暴力破解锁定** — 密码连续输错5次，系统锁定30秒，期间无法输入密码
- **自动恢复** — 锁定到期后自动重置失败计数，重新开始计数
- **微信实时告警** — 锁定触发时自动推送微信通知，提醒管理员注意
- **统一重置** — 任何方式成功开锁（RFID/指纹/密码/蓝牙）都会清除安全状态
- **黑名单封禁** — 丢失的RFID卡或指纹可加入黑名单，刷卡/放手指时直接拒绝并拍照
- **失败次数提示** — 密码输入界面显示剩余尝试次数，提醒用户注意

## 六、快速上手

### 1. 烧录主 ESP32

Arduino IDE 打开 `smart_door_lock/smart_door_lock.ino`，设置：
- 开发板: ESP32 Dev Module
- 分区方案: Huge APP (3MB No OTA/1MB SPIFFS)（必须选这个，否则代码放不下）
- 上传速度: 921600

点击上传。如果报错 "Write failed" 或 "MD5 mismatch"，先执行擦除：
```
python -m esptool --chip esp32 --port COM4 erase-flash
```
然后重新上传。

### 2. 上传 ESP32-CAM

用 Thonny IDE 连接 ESP32-CAM，将 `模块资料/cam.py` 上传到设备根目录，重启运行。

### 3. 启动服务器

```
pip install flask werkzeug
python server.py
```

访问 http://127.0.0.1:8000
- 登录账号: admin
- 登录密码: 123456

### 4. 安装蓝牙 App

将 `SmartLockPro_md3_unlock_only.apk` 传到安卓手机，安装后打开使用。

### 5. 首次配置

1. 打开网页后台 → **白名单管理** → 添加你的 RFID 卡号和指纹 ID
2. 打开 **密码设置** → 修改开门密码（默认 1234）
3. 打开 **通知设置** → 填入 Server酱 SendKey 实现微信推送

## 七、使用方法

### 刷卡开锁
将已录入白名单的 RFID 卡靠近 RC522 模组，蜂鸣器滴一声，门锁打开，5秒后自动关锁。

### 指纹开锁
将已录入的手指放到 ZW101 模组上，模组自动唤醒识别，通过后蜂鸣器滴一声，门锁打开。

### 密码开锁
1. 按键盘任意键进入密码输入界面
2. 输入4位数字密码（OLED 显示 * 号）
3. 按 # 键确认
4. 右侧按键: A=删除, B=清空, C=确认, D=返回主界面

### 蓝牙开锁
1. 打开手机蓝牙，打开 SmartLockPro App
2. 点击扫描，选择 "SmartLock" 设备
3. 输入密码，点击开锁

### 网页远程开锁
登录 Web 管理后台，点击远程开锁按钮。

### 查看验证记录
登录 Web 管理后台 → 验证记录，查看所有开锁/失败记录，含照片、时间、验证方式。

### 导出记录
点击验证记录页面的导出按钮，下载 CSV 文件。

## 十一、数据库设计

系统使用 SQLite 数据库（access.db）存储所有业务数据，包含 5 张表：

| 表名 | 用途 | 主要字段 |
|------|------|---------|
| logs | 门禁验证记录 | id, filename(照片), timestamp, method(方式), result(结果), size |
| whitelist | 白名单（授权访问） | id, type(rfid/fp), value(卡号/指纹ID), name(备注), created_at |
| blacklist | 黑名单（封禁访问） | id, type(rfid/fp), value(卡号/指纹ID), name(备注), created_at |
| device_status | 设备实时状态 | lock_state(锁状态), cam_online(摄像头在线), last_update |
| settings | 系统配置 | key, value（密码、配置版本、Server酱Key、通知事件类型等） |

### 数据流

```
ESP32 验证 → ESP32-CAM 拍照上传 → Flask 接收 → 写入 logs 表 + 保存照片
管理员操作 → Web UI → Flask API → 更新 whitelist/blacklist/settings 表
ESP32 轮询 → GET /api/config → 读取 whitelist/blacklist/settings → 同步到本地
```

### 数据库操作

- 所有写操作自动触发 config_version 递增，ESP32 下次轮询时自动同步
- 照片存储在 uploads/ 目录，logs 表记录文件名和大小
- 支持 CSV 导出全部验证记录

## 十三、注意事项

1. ESP32-CAM 必须使用独立 5V 电源供电，不能从主 ESP32 取电
2. WiFi 名称、密码、服务器 IP 需在代码中配置，与局域网匹配
3. 默认开门密码为 1234，建议尽快修改
4. 网络环境变化（换路由器/重启）后 IP 可能改变，需同步更新代码中的 SERVER_HOST
5. Arduino IDE 上传时分区方案必须选 "Huge APP (3MB)"，否则代码超出 Flash 空间
6. 上传失败时先执行 `python -m esptool --chip esp32 --port COMx erase-flash` 再重试
7. ESP32-CAM 的 UART 使用 GPIO1/GPIO3，与串口监视器冲突，这是硬件限制
8. 蓝牙和 WiFi 共用 2.4GHz 射频，可能互相影响，代码中已做兼容优化
