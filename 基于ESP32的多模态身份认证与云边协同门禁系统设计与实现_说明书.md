# 基于ESP32的多模态身份认证与云边协同门禁系统设计与实现

---

## 1. 项目概述

本系统基于ESP32嵌入式开发平台，融合RFID识别、指纹识别、密码输入、蓝牙开锁、ESP32-CAM图像采集以及云端数据同步技术，设计并实现一套多模态身份认证智能门禁系统。

系统通过四种身份验证方式实现门禁控制，并结合物联网技术实现远程数据记录、行为追踪与安全管理，提升门禁系统的安全性与可管理性。

---

## 2. 系统总体功能

本系统采用**三层架构**设计，分为设备执行层、数据传输层和管理控制层：

**设备端（ESP32主控 + ESP32-CAM）—— 认证执行层：**
- RFID身份识别（RC522模块）
- 指纹身份识别（ZW101半导体指纹模组）
- 矩阵键盘密码验证（4x4键盘）
- 经典蓝牙开锁（手机App）
- SG90舵机门锁控制
- SSD1306 OLED状态显示（128x64）
- 蜂鸣器声音反馈
- ESP32-CAM拍照记录（验证成功/失败均拍照）
- Flash持久化存储（断电不丢失配置）
- 离线日志缓存与联网自动补传

**数据传输层（WiFi + HTTP）：**
- ESP32每10秒从服务器轮询配置（密码、白名单、黑名单）
- ESP32-CAM通过UART接收指令，拍照后HTTP上传至服务器
- 云端配置版本号机制，增量同步避免全量拉取

**管理端（Flask Web后台）—— 管理控制层：**
- 验证记录查看与CSV导出
- 白名单管理（RFID卡号 + 指纹ID）
- 黑名单管理（封禁丢失的卡/指纹）
- 通知设置（Server酱微信推送）
- 密码修改（在线修改，自动同步至ESP32）
- 远程开锁（网页一键开门）
- 设备状态监控（门锁状态、摄像头在线状态）

## 3. 功能模块说明

---

### 3.1 RFID身份识别模块

**硬件：** MFRC522射频模块，SPI接口

**功能描述：** 通过RFID模块读取用户卡号UID，与白名单比对实现身份识别。

**引脚连接：**
| RC522引脚 | ESP32引脚 |
|-----------|-----------|
| SDA (SS) | GPIO 5 |
| SCK | GPIO 18 |
| MOSI | GPIO 23 |
| MISO | GPIO 19 |
| RST | 不接（内部上电复位） |

**功能流程：**
1. RFID模块持续检测卡片靠近
2. 读取卡片UID（4字节）
3. 先查黑名单，命中则直接拒绝并拍照
4. 再查白名单，命中则验证通过
5. 未命中任何名单则视为未知卡片，拒绝

**结果输出：**
- 通过：蜂鸣器滴一声，舵机开门
- 黑名单命中：蜂鸣器滴三声，OLED显示拒绝，触发拍照
- 未知卡片：蜂鸣器滴三声，触发拍照

---

### 3.2 指纹识别模块

**硬件：** ZW101半导体指纹模组，UART通信（波特率57600）

**功能描述：** 通过指纹模块进行生物识别验证，模组自带RGB三色灯指示状态。

**引脚连接：**
| ZW101引脚 | ESP32引脚 | 说明 |
|-----------|-----------|------|
| TX | GPIO 16 | 模组发→ESP32收 |
| RX | GPIO 17 | ESP32发→模组收 |
| VCC | 3.3V | 供电 |
| V_SENSOR | 3.3V | 常供电（手指唤醒） |

**功能流程：**
1. 每3秒轮询一次指纹识别（非阻塞）
2. 手指放到模组上，模组自动唤醒采集指纹
3. 模组内部完成特征提取与比对（1:N搜索）
4. 识别成功返回匹配ID，先查黑名单再查白名单
5. 识别失败（未匹配）返回失败码

**LED指示：**
- 蓝色呼吸灯：待机状态
- 绿色常亮：识别通过
- 红色闪烁：识别失败

---

### 3.3 矩阵键盘密码模块

**硬件：** 4x4矩阵键盘

**引脚连接：**
| 键盘引脚 | ESP32引脚 |
|---------|-----------|
| R1-R4 | GPIO 26, 25, 33, 32 |
| C1-C4 | GPIO 27, 14, 13, 3 |

**功能描述：** 提供密码输入方式进行身份验证，支持防暴力破解锁定。

**键盘布局：**
```
1  2  3  A(删除)
4  5  6  B(清空)
7  8  9  C(确认)
*  0  #  D(返回)
```

**功能流程：**
1. 按任意键进入密码输入界面
2. 输入4位数字密码（OLED显示*号）
3. 按#或C确认提交
4. 密码正确→开锁；错误→提示失败
5. D键返回主界面

**防暴力破解：**
- 连续输错5次，系统锁定30秒
- 锁定期间OLED显示倒计时
- 锁定到期自动重置失败计数
- 触发时自动发送微信告警通知
- RFID/指纹成功开锁会清除失败计数
- 蓝牙输错也计入失败次数

---

### 3.4 蓝牙开锁模块

**硬件：** ESP32经典蓝牙SPP（BluetoothSerial）

**功能描述：** 手机App通过蓝牙连接ESP32，发送密码实现远程开锁。

**功能流程：**
1. ESP32启动蓝牙，设备名"Smart_Lock"
2. 手机App扫描并连接
3. App发送4位密码
4. ESP32验证密码，返回OK或FAIL
5. 验证通过则开锁

**配套App：** MIT App Inventor开发的SmartLockPro App（APK已包含在项目中）

---

### 3.5 门锁控制模块

**硬件：** SG90舵机

**引脚：** GPIO 15

**控制逻辑：**
- 锁定角度：85°
- 开锁角度：180°
- 自动关锁时间：5秒（可配置）

---

### 3.6 OLED显示模块

**硬件：** SSD1306 128x64 I2C OLED，U8g2驱动

**引脚：** SDA=GPIO 21, SCL=GPIO 22

**显示界面：**

| 界面 | 内容 |
|------|------|
| 主界面 | 标题栏(智能门锁+WiFi/BT状态+锁图标) + 大字时间(logisoso16) + 年月日星期 + 底部三列(刷卡/指纹/密码) |
| 密码输入 | 顶部栏(返回+输入密码) + 密码框(*号) + 失败次数提示(WARN:Xleft) + 4x4虚拟键盘 |
| 验证通过 | "解锁成功" + 锁图标(打开) + "已开锁" + "5秒后自动关锁" |
| 验证失败 | "验证失败" + X图标 + "拒绝访问" + "密码错误" |
| 已开锁 | "解锁成功" + 锁图标 + "已开锁" + 倒计时秒数 |
| 安全锁定 | "安全锁定" + 锁图标(锁定) + 倒计时 + "密码连续错误已锁定" |

---

### 3.7 蜂鸣器模块

**引脚：** GPIO 4（高电平触发）

**声音反馈：**
| 事件 | 声音 |
|------|------|
| 验证通过 | 滴一声（100ms短促） |
| 验证失败 | 滴三声（连续短促） |

---

### 3.8 ESP32-CAM拍照记录模块

**硬件：** ESP32-CAM（MicroPython固件，PSRAM版）

**通信方式：** UART串口（TX=GPIO1, RX=GPIO3，波特率115200）

**功能描述：** 每次验证尝试（成功或失败）自动拍照并上传至Flask服务器。

**UART指令协议：**
| 指令 | 格式 | 功能 |
|------|------|------|
| PING | `PING` | 测试连接，返回`PONG` |
| CAPTURE | `CAPTURE\|方式\|结果` | 拍照并上传，返回`OK`或`FAIL` |
| STATUS | `STATUS\|状态` | 上报门锁状态 |

**工作流程：**
1. 主ESP32通过UART发送CAPTURE指令
2. ESP32-CAM拍摄JPEG照片（QVGA分辨率）
3. 通过HTTP multipart POST上传至Flask服务器
4. 服务器保存照片并写入数据库

**心跳机制：** 每30秒向服务器发送心跳，服务器超过60秒未收到则标记离线

## 4. 数据存储设计

### 4.1 服务端SQLite数据库

Flask服务器使用SQLite数据库（access.db），包含5张表：

| 表名 | 用途 | 主要字段 |
|------|------|---------|
| logs | 门禁验证记录 | id, filename(照片), timestamp, method(方式), result(结果), size(大小) |
| whitelist | 白名单 | id, type(rfid/fp), value(卡号/指纹ID), name(备注), created_at |
| blacklist | 黑名单 | id, type(rfid/fp), value(卡号/指纹ID), name(备注), created_at |
| device_status | 设备状态 | lock_state(锁状态), cam_online(摄像头在线), last_update |
| settings | 系统配置 | key, value（密码、配置版本、Server酱Key、通知事件类型等） |

### 4.2 ESP32端Flash持久化

ESP32使用Preferences库（NVS分区）存储配置，断电不丢失：

| 键名 | 数据类型 | 内容 |
|------|---------|------|
| password | bytes | 4位开锁密码 |
| rfidCount | uchar | RFID白名单数量 |
| rfidList | bytes | RFID白名单数据 |
| fpCount | uchar | 指纹白名单数量 |
| fpList | bytes | 指纹白名单数据 |
| rfidBlCount | uchar | RFID黑名单数量 |
| rfidBlackList | bytes | RFID黑名单数据 |
| fpBlCount | uchar | 指纹黑名单数量 |
| fpBlackList | bytes | 指纹黑名单数据 |
| configVer | uint | 配置版本号 |

**启动流程：** 上电先从Flash读取 → 联网后用服务器最新数据覆盖 → 写回Flash

### 4.3 离线日志缓存

ESP32维护一个内存缓冲区（最多20条），用于断网时缓存验证记录：

- 每次验证（成功/失败）调用 `addOfflineLog()` 存入缓冲区
- 联网后在 `pollConfig()` 成功同步时自动调用 `uploadOfflineLogs()` 补传
- 补传成功则移除已上传记录，失败则保留等待下次重试
- 缓冲区满时丢弃最早记录

## 5. 系统工作流程

### 5.1 系统初始化阶段
1. 系统上电，从Flash加载上次同步的配置（密码、白名单、黑名单）
2. 初始化各硬件模块（OLED、WiFi、RFID、舵机、键盘、指纹、ESP32-CAM、蓝牙）
3. WiFi连接成功后，每10秒从服务器轮询最新配置
4. NTP同步实时时钟

### 5.2 待机监测阶段
系统循环监测四种输入方式，用户可任选一种：
- **RFID**：卡片靠近自动感应
- **指纹**：每3秒轮询，手指放上自动识别
- **密码**：按任意键进入输入界面
- **蓝牙**：手机App发送密码

### 5.3 验证成功流程
1. OLED显示"解锁成功"
2. 蜂鸣器滴一声
3. LED亮起
4. 舵机转动至开锁角度
5. ESP32-CAM拍照上传
6. 记录写入离线日志缓冲区
7. 5秒后自动关锁

### 5.4 验证失败流程
1. OLED显示"验证失败"
2. 蜂鸣器滴三声
3. LED闪烁
4. ESP32-CAM拍照上传（标记FAIL）
5. 记录写入离线日志缓冲区
6. 密码失败时failCount+1，达5次触发30秒锁定

- Session登录机制，未登录自动跳转

### 6.2 功能标签页

| 标签页 | 功能 |
|--------|------|
| 验证记录 | 查看全部记录（时间、方式、结果、照片缩略图），支持删除、清空、CSV导出 |
| 白名单管理 | 添加/删除RFID卡号和指纹ID，支持备注名称 |

| 通知设置 | 配置Server酱SendKey，独立开关各事件通知（成功/失败/远程/摄像头） |
| 密码设置 | 修改开门密码，含强度指示、确认匹配、可见性切换 |

### 6.3 API接口

| 路由 | 方法 | 功能 |
|------|------|------|
| `/api/config` | GET | ESP32轮询配置（密码+白名单+黑名单+远程开锁） |
| `/api/config` | POST | 修改密码，触发配置版本递增 |
| `/api/whitelist` | GET/POST/DELETE | 白名单CRUD |
| `/api/blacklist` | GET/POST/DELETE | 黑名单CRUD |
| `/api/status` | GET/POST | 设备状态读写（门锁状态、摄像头在线） |
| `/api/unlock` | GET | 远程开锁 |
| `/api/unlock/done` | GET | 远程开锁确认 |
| `/api/alert` | POST | ESP32发送暴力破解告警 → 推送微信 |
| `/api/log` | POST | 离线日志补传 |
| `/api/notify-settings` | GET/POST | 通知配置读写 |
| `/api/notify-test` | GET | 测试微信推送 |
| `/api/export/csv` | GET | 导出验证记录CSV |
| `/upload` | POST | ESP32-CAM上传照片 |
| `/photos` | GET | 获取全部记录列表 |
| `/photo/<filename>` | GET | 获取照片文件 |

### 6.4 微信通知
- 接入Server酱（sct.ftqq.com）
- 支持按事件类型独立开关：开锁成功、开锁失败、远程开锁、摄像头抓拍
